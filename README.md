## Commit Convention
| commit   | 사용 예시         |
| -------- | ---------------- |
| feat     | 새로운 기능 추가  |
| refactor | 리팩토링          |
| fix      | 버그 수정         |
| design   | CSS 스타일링      |
| docs     | 문서 수정         |
| remove   | 파일 삭제         |
| chore    | 패키지 매니저 수정 |
| deploy   | 배포              |

## Branch 전략
| branch   | 사용 예시                                  |
| -------- | -----------------------------------------|
| main     | 서비스를 직접 배포하는 브랜치                   |
| feature  | 각 기능별 개발 브랜치                         |
| develop  | 개발된 내용이 저장되는 브랜치                   |
| release  | 배포 전 품질 검사를 위한 브랜치                 |
| hotfix   | main 브랜치에 배포 후 버그 발생 시 사용하는 브랜치  |
